# Tytuł projektu: [Nazwa projektu] #
## Cel: [Krótki opis celu projektu] ##

- Inspiracje (słowa, skojarzenia, pomysły) 
   - prostota, szybkie zadania, minimalizm UX
   - automatyzacja rutynowych czynności
   - personalizacja, profile użytkownika
   - micro-interactions, natychmiastowy feedback
   - offline-first, synchronizacja w tle
   - gamifikacja: odznaki, postęp, małe cele
   - „zero friction” onboarding
   - privacy by design, kontrola danych
   - integracje: kalendarz, powiadomienia, API zewnętrzne
   - lokalizacja: PL, wielojęzyczność

 - Krótka lista elementów projektu (kategorie / moduły) 
   - Autoryzacja / Profile
   - Zarządzanie zadaniami / Lista priorytetów
   - Kalendarz / Terminy i przypomnienia
   - Integracje (np. Google Calendar, Slack)
   - Raporty / Statystyki użycia
   - Ustawienia prywatności i eksport danych
   - System powiadomień (push, email)

 - Wstępne dane wejściowe (przykłady) 
   - lista kategorii: Praca, Dom, Edukacja, Hobby, Zakupy
   - lista funkcji: dodaj zadanie, przypomnienie, powtarzanie, tagi, filtrowanie
   - role użytkowników: właściciel, współpracownik, obserwator

 - Pomysłowe funkcje / eksperymenty 
   - tryb „szybkiego dodawania” z sugerowanymi terminami
   - inteligentne priorytetyzowanie na podstawie kontekstu (wyjazd, deadline)
   - skróty klawiaturowe i automatyzacje (makra)
   - widżet mobilny z jednym tapnięciem do dodania zadania
   - API do integracji z narzędziami zespołowymi

 - Lista źródeł pomysłu 
   - własna potrzeba: szybkie zarządzanie krótkimi zadaniami
   - feedback znajomych/zespołu: brak prostych narzędzi „na dziś”
   - analiza konkurencji: uprościć to, co zbyt rozbudowane u innych
   - przypadek: brak synchronizacji offline powoduje utratę danych

 - Notatki / Zasady projektowe 
   - Rozbij duże zadania (>24h) na podzadania.
   - Priorytety: W (wysoki), S (średni), N (niski).
   - Aktualizuj status: Do zrobienia / W trakcie / Zrobione.
   - Raz w tygodniu synchronizacja statusów i przegląd priorytetów.
   - Zapisuj kluczowe decyzje i zmiany przy każdym zadaniu.
   - Prototypuj szybko: MVP w 2–4 tygodnie, iteracje co sprint.

 - Przykładowe pomysły na nazwę (branding) 
   - SzybkaPracka
   - TaskBlink
   - DziśProsto
   - MiniSprint
